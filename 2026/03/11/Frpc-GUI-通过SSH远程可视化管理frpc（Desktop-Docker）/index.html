我做了一个 frpc 控制台：连接、改配置、看日志、重启，一条链路闭环

发表于 2026-03-11 分类于 联创项目 ， Web开发 ， 实用工具 阅读次数： 本文字数： 813 阅读时长 ≈ 3 分钟

Frpc-GUI：通过 SSH 远程可视化管理 frpc（Desktop / Docker）

告别 SSH + nano：用 Frpc-GUI 管理多台服务器上的 frpc

摘要
Frpc-GUI 是一个现代化面板，用于通过 SSH 远程管理目标机器上的 frpc：保存多服务器连接、可视化管理 proxies、编辑配置、查看状态与日志，并在需要时重启服务。适合 frpc 部署在路由器 / NAS / VPS / 小主机等场景。
链接：

  • GitHub： https://github.com/dukaworks/frpc-gui
  • Releases： https://github.com/dukaworks/frpc-gui/releases
  • Docker 镜像： ghcr.io/dukaworks/frpc-gui:latest

背景：frpc 的“日常运维摩擦”

frpc 很稳定，但“改配置”这件事并不轻松，尤其当它跑在路由器/NAS/VPS 上时：

  • 改一次代理：SSH 上去找配置文件（TOML/INI）→ 手改 → 重启 → tail 日志
  • 多台机器：靠记忆/笔记切来切去，容易弄错目标机或配置文件路径
  • 改完验证：状态、日志、端口连通、域名访问……经常要来回折腾
    我希望把这套流程变得更像“控制台”： 连接 → 编辑 → 保存 → 验证 ，并且能方便地管理多台服务器。

Frpc-GUI 是什么

Frpc-GUI 是一个用于远程管理 frpc 的可视化控制台。

  • 控制端运行在你的电脑（桌面版）或一台机器上（Docker/Web）
  • 通过 SSH 连接到目标机
  • 对目标机上的 frpc 做常见运维动作：读/写配置、重启、看状态和日志

特性概览

  • SSH 远程连接 ：连接任意运行 frpc 的服务器/路由器/NAS
  • 多服务器档案 ：保存多套 SSH 连接配置，支持列表分页
  • 可视化管理 proxies ：新增/编辑/删除，支持批量操作
  • 配置模式 + 源码模式 ：兼顾新手和熟练用户
  • 状态与日志 ：查看状态、抓日志、按需重启（Docker/Systemd/进程等场景）
  • 桌面版外链体验更合理 ：FRPS URL 这类地址会用系统默认浏览器打开，更适合登录/授权流程
  • 双语界面 ：English / 中文（i18n）

快速开始

方式一：桌面版（推荐）

从 GitHub Releases 下载对应平台安装包：

  • https://github.com/dukaworks/frpc-gui/releases
    包含：
  • Windows： .exe installer + .zip
  • macOS： .dmg
  • Linux： .deb

启动后：

  1. 填 SSH 连接信息并保存
  2. 连接目标机
  3. 在 Proxies/Config/Logs 页面完成配置与验证

方式二：Docker

Docker Compose：

docker compose up -d

访问：

  • http://localhost:3000
    Docker Run：
docker run -d \
  --name frpc-gui \
  -p 3000:3000 \ 
  ghcr.io/dukaworks/frpc-gui:latest

工作原理（简述）

  • 你在本地（桌面版）或浏览器（Docker/Web）打开 Frpc-GUI
  • Frpc-GUI 通过 SSH 在目标机上执行：读取/写入配置、重启、抓日志、状态检查等操作
  • 桌面版内置服务绑定 127.0.0.1 ，减少暴露面
