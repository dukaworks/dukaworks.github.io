SSH 远程连接

发表于 2026-03-04 分类于 实用工具 阅读次数： 本文字数： 348 阅读时长 ≈ 1 分钟

SSH（Secure Shell）是远程登录 Linux 服务器的标准协议，提供加密的通信通道，安全可靠。

基本连接、ssh user@host 连接到远程主机，首次连接会提示保存主机密钥。ssh user@host -p 2222 指定端口（默认 22）。ssh -i key.pem user@host 使用密钥文件认证。

密钥认证比密码更安全、更方便。生成密钥、ssh-keygen -t rsa -b 4096。公钥复制到服务器、ssh-copy-id user@host。之后登录无需密码。

SSH 端口转发可以实现内网穿透。本地端口转发、ssh -L local_port:remote_host:remote_port user@jump_host。远程端口转发、ssh -R remote_port:local_host:local_port user@jump_host。

SSH 配置文件 ~/.ssh/config 简化连接。定义 Host 别名、用户名、主机、端口、密钥等，之后直接 ssh alias 连接。

SSH 命令执行、ssh user@host "command" 在远程主机执行命令。ssh user@host "tar -czf - /data" > backup.tar.gz 远程打包传回本地。

SSH 隧道可以实现安全的远程访问。例如，访问远程数据库、ssh -L 3306:localhost:3306 user@host，然后本地连接 localhost:3306 就是远程数据库。

SSH 还支持文件传输、scp local_file user@host:/remote/path 上传文件，scp user@host:/remote/path local_file 下载文件。rsync 更适合大量文件同步。

SSH 是服务器运维的基础工具。掌握 SSH 能让你轻松管理远程服务器，是 Linux 用户的必备技能。
