Docker Compose 使用指南

发表于 2026-03-04 分类于 实用工具

Docker Compose 是 Docker 官方提供的容器编排工具，通过 YAML 文件定义和运行多容器应用，简化了开发和部署。

Compose 文件（docker-compose.yml）定义服务、网络、卷。基本结构、

常用命令、docker-compose up 启动服务，-d 后台运行，--build 强制重新构建。docker-compose down 停止并删除容器、网络。docker-compose logs 查看日志，-f 实时跟踪。

服务定义、image 指定镜像，build 指定构建上下文，ports 暴露端口，volumes 挂载卷，environment 设置环境变量，depends_on 定义依赖关系。

网络管理、Compose 自动创建默认网络，服务可以通过服务名相互访问。可以自定义网络，使用 external 链接外部网络。

卷管理、volumes 声明命名卷，在服务中挂载。卷在容器删除后仍然保留，用于持久化数据。可以挂载主机目录到容器。

环境变量、直接在 docker-compose.yml 中定义，或使用 .env 文件。${VAR_NAME} 语法引用变量，方便不同环境使用不同配置。

扩展服务、docker-compose scale web=3 扩展服务实例数量。配合负载均衡器，实现简单的水平扩展。

开发场景、Compose 非常适合本地开发环境。一键启动所有服务（数据库、缓存、应用），环境一致，团队共享。

虽然 Kubernetes 功能更强大，但对于简单场景，Compose 足够用。学习成本低，使用简单，是开发者的得力助手。
