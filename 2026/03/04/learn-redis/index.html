Redis 实战指南

发表于 2026-03-04 分类于 学习探索

Redis 是高性能的键值存储系统，常用于缓存、消息队列、排行榜等场景。丰富的数据类型和出色的性能使其成为开发者的首选。

基本数据类型、String（字符串）、Hash（哈希）、List（列表）、Set（集合）、ZSet（有序集合）、Bitmap（位图）、HyperLogLog（基数统计）。

String 类型、最基本的数据类型，可以存储字符串、数字。常用命令、SET、GET、INCR、DECR、EXPIRE。应用场景、计数器、分布式锁、简单缓存。

Hash 类型、适合存储对象。命令、HSET、HGET、HMGET、HGETALL。相比 String 存储对象的 JSON，Hash 可以单独修改字段，效率更高。

List 类型、有序列表，适合队列和栈。命令、LPUSH、RPUSH、LPOP、RPOP、LRANGE。应用场景、消息队列、最新列表、时间线。

Set 类型、无序唯一集合，适合去重。命令、SADD、SMEMBERS、SINTER、SUNION。应用场景、标签系统、共同好友、去重。

ZSet 类型、有序集合，每个元素有分数。命令、ZADD、ZRANGE、ZRANK。应用场景、排行榜、延时队列、范围查询。

Redis 持久化、RDB 快照、AOF 日志、混合持久化。RDB 适合备份，AOF 更安全，混合兼顾两者。

Redis 集群、主从复制、哨兵模式、Cluster 集群。主从复制实现读写分离，哨兵提供高可用，Cluster 实现水平扩展。

性能优化、选择合适的数据类型，使用 Pipeline 批量操作，避免大 key，合理设置过期时间，监控内存使用。

Redis 简单易用但功能强大。掌握其特性，能解决很多实际问题。是后端工程师必备技能。
