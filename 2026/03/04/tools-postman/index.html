Postman 使用指南

发表于 2026-03-04 分类于 实用工具 阅读次数：

Postman 是 API 开发和测试的必备工具。它提供了直观的界面，简化了 API 调用和调试过程。

发送请求是 Postman 的基本功能。选择 HTTP 方法（GET、POST、PUT、DELETE 等），输入 URL，添加请求头、请求体，点击发送即可看到响应结果。响应内容包括状态码、响应头、响应体等。

环境变量让测试更灵活。可以为不同的环境（开发、测试、生产）设置不同的变量值，如基础 URL、认证信息等。使用 {{variable_name}} 语法引用变量。

集合（Collection）可以组织相关的请求。可以给集合添加变量、前置脚本、测试脚本，实现自动化测试。可以导出分享给团队成员，确保测试标准统一。

测试脚本可以验证响应结果。Postman 提供了丰富的断言函数、pm.test() 定义测试，pm.expect() 断言值，pm.response.to.have.status() 检查状态码。可以检查数据格式、业务逻辑等。

Newman 是 Postman 的命令行工具，可以将集合集成到 CI/CD 流程中，实现自动化测试。这对于保证 API 质量和持续集成非常有用。

Postman 还提供了 Mock Server 功能，可以根据接口定义生成模拟数据，方便前后端并行开发。

掌握 Postman 不仅能提高 API 开发效率，还能提升测试质量和团队协作水平。
